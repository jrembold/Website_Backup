The Setup

I have included the pgl.py library in the repository for today, and will generally do so for any lab or assignment that needs it. Know that if you ever want to play around with it, you just need to copy that file into the same directory as whatever script it is that you are wanting to use it. In order to not have issues with VSCode finding the file, I recommend doing one of the following: * Always open the full folder when you are working on a project, or * Click the gear in the lower left and select “Settings”. Then search for python execute. The third option down should be a checkbox that says Python > Terminal: Execute in File Dir. Check that box, and it should ensure that VSCode uses the file directory even if you opened just the file by its lonesome.

Hints

  • You can draw the tire either using two ovals or a very thick line.

  • You can adjust the line width of an object with .set_line_width(value)

  • Positioning is often times tough without being able to visualize the pixel grid. So maybe consider drawing a temporary grid to help you! You can always comment those lines out later. For example, you can draw a bunch of vertical lines using something like:

    sep = 50
for c in range(0,GW_WIDTH,sep):
    gw.add(GLine(c, 0, c, GW_HEIGHT))

    You can figure out the horizontal lines. Just make sure you include the grid lines after anything else you add so that they will be on the top, and remove them (or comment them out) before you upload your final code!
