Today, you will be building atop an already existing piece of code. In the template is a piece of code for constructing a simple clicky box game, wherein a box appears at a random location on the screen and, upon being clicked on, moves to a different location. Your task today will be to spruce up the game with a bit of animation. Learning goals will focus on:

Part 2)

You will not be able to use the existing “click” callback to create the new box. The reason is because the first thing that happens after you click is that the clicked box shrinks, and THEN you want to create the new box. As a result, you will need to move the logic for creating the new box over to the animating step function. Don’t get rid of your “click” callback though! The difference now is that, when you click on the box, you want to indicate to the program that the box is supposed to start shrinking. To do this, you need to communicate with the step() function, which means you’ll need to use GState to pass the information between them. Right after you create the original box, add a GState variable called gs.is_shrinking which you can initially set to False. This is a little boolean “flag” that you’ll use to communicate to your step function whether the box should be shrinking or not. Now, what should happen when the square is clicked? Make the necessary fix and then move on to the step callback.

Part 3)

So now everything needs to happen in the step function, which means you need to think through what all the desired things that you could want the program to be doing. Here, you basically have 3 things to account for:

  • If the box has shrunk to nothing (its size is less than 0), you want to return the box to its original size and move it to a random part of the screen. This will look very similar to parts of the original “click” callback, just don’t forget to also bring the box back to its original size.
  • If is_shrinking is currently True, then you need to be shrinking the box, which you can do by changing its width and height by a small amount each time. In order for the box to appear to shrink towards its center point, you will also need to adjust the box position a little bit each time. You can do all of this at the same time with the .set_bounds() method, in which you specify the x position, y position, width, and height (so very similar to what you specify when you first created the box). Or you can do it in two steps, first setting the size and then moving the box appropriately.
  • If the box is not supposed to be shrinking and has a normal size, don’t do anything!
