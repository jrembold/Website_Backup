Lab 18: Logo Bounce

April 9, 2021

Today we’ll focus on continuing to build our familiarity and capabilities when working with classes, in particularly when inheriting from GCompounds. Learning objectives will include:

  • Extending GCompound with appropriate __init__ functions
  • Creating and working with a new instance of our class
  • Utilizing methods to build convenience features directly into the class itself.

I’ve already written a fair bit of code for you in the starting repository, so make sure you read it over and take advantage of it! You can add the lab Github repository by going here.

Instructions

The opening scene of The Office “Launch Party” episode begins with everyone eagerly waiting for a small bouncing logo to perfectly find it’s way into a corner. You can watch the scene here (sorry, Youtube wouldn’t let me embed it). Our goal today is to recreate that bouncing logo. How much time you spend watching it and waiting for it to bounce perfectly into a corner is up to you!

Part 1)

The logo itself is simple, and is comprised simply of a square and two text elements. To hasten things along and not let you get bogged down in small positionings, I have already included code to create the logo for you in the template. However, that code is laid out as we would have previously achieved this, with a separate function to handle creating the GCompound. Here, you want to define a new class called DVDLogo that will inherit from GCompound. You should add a constructor function to your class which will take no arguments other than self, and then make sure you call GCompound's constructor function as the first line. You can then move most of the existing code to create the logo into the constructor function (__init__) in your new class.

The code can almost exist exactly at is was in the original create_object function, with some small changes. Because your class is inheriting from GCompound, any reference to the original obj should be changed instead to reference self. Additionally, you may have noticed in the video that the color of the logo square changes each time it bounces. In order for us to duplicate this, you should keep track of the reference to that particular GRect, usually by creating an attribute and setting it equal to the GRect. Otherwise, as soon as the constructor function returns, we lose all information that was not saved as an attribute.

Switch the code at the bottom to assign logo to an instance of your newly created class, and then make sure it is appearing on the screen again. It should look just like the original. All we’ve really accomplished so far is moving code around.

Part 2)

Now we can start adding some functionality! To begin, an easy option. Add a method called color_change which takes no arguments other than self. When color change is called, it should simply change your rectangle’s color to a new random color. How could you test this? Go ahead and do so to ensure you have implemented things properly before continuing.

Part 3)

Now to get things moving! Add two more attributes to your class’s constructor function called self.vx and self.vy. These should be set to a random number between 1 and 5. Then add a new method to your class called update, which again takes no arguments besides self. Whenever update is called, you should call the move method that exists for all GObjects and move your custom object vx pixels horizontally and vy pixels vertically. To get things animated, add an interval timer at the end of your code, where the callback function you want to call will be logo.update, which is the method that you just defined. Now if you run your code, the logo should start moving!

Part 4)

Finally, you just need to add collision detection. You did this in breakout, so it should be simple enough. In your update method, after you move the object, check to see if any part of the object has moved off the screen. If it has, reverse the needed velocity attribute (vx or vy) so that it bounces off the wall. In addition, call your color_change method whenever it hits the wall so that your logo will change colors each bounce. And that should be all! Now you can happily watch your logo bounce around the screen!
