Instructions

Our goal is to simulate the playing of the game Red Light, Green Light with several players. We will create a class object called Runner which will hold the needed information about each individual player. And then we will write a short script to bring it all together and simulate the game.

Part 1)

The runner class will need to keep track of a handful of attributes. Each of our runners will have a name, as well as a rate of acceleration and a top speed. Those three values will all be passed in as arguments upon creation of a Runner object. All runners will also have a current speed and position that the class should keep track of. These will start at 0 for all runners. Write the appropriate __init__ method to get all of this set up.

We are going to create and use two class variables in this problem. One of these will be the current status of the red light or green light, telling the runners whether to go or not. Call this class variable go and set it initially equal to False. This makes sense to be a class variable since when we change it, we want it to change equally across all runners simultaneously. The other class variable we will create will be called goalline and will set where the final goal is located for the runners. This makes sense to be a class variable since it will be constant across all runners. Go ahead and implement both these class variables, setting goalline to be 80.

Part 2)

We will add a few useful methods to our runner class now. First off, add a method called update_position which only accepts self. This function will be responsible for implementing the runner's acceleration, speed, and top-speed attributes to compute their new position. These updates should ONLY occur if the light is currently Green (aka, go is True). A common way you can update speed and position is as follows:
speed=speed+accelerationspeed = speed + accelerationspeed=speed+acceleration
and then,
position=position+speedposition = position + speedposition=position+speed

Once the speed is updated, you should check to make sure it is not greater than that runner's top-speed, and set it back to the top-speed if it is. The other requirement should be that if the light is Red, (go is False), then the speed should be set back to 0.

The two other methods to implement are easier:

  • draw(self): This method will be responsible for printing to the screen a graphical depiction of the current position of the runner. I padded the space already traveled with dashes, so for a single runner with name 'A' at position 15, this might look like:
---------------A
  • is_winner(self): checks to see if the runner's position is greater than the goal-line and returns True if it is.

Part 3)

Now we are positioned to put this class to use! Follow the general steps below to write a script to play Red Light, Green Light:

  • Create a list of 3 Runner objects with the properties and names given by the following table:

    Name Acceleration Top Speed
    A 3 8
    B 6 6
    C 4 7

  • Write a loop that will keep going as long as there is no winner. Each iteration,

    • Ask the user to input the new state of the light (R or G?)
    • Update the class variable go according to the input
    • Loop through each runner object and:
      • Update their position
      • Draw them to the screen
      • Check if they are a winner
    • If a winner is found, print a victory message to the screen and end the loop!

Run it and play through a few times! Can you make each of the players win by a clever choice of Red lights and Green lights?
