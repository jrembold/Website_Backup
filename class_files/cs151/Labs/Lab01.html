Lab 1: Github and Github Classroom

Step 1: Getting a Github account if you don't already have one

Getting setup on Github is pretty straightforward:

  1. Navigate you way to www.github.com and click on "Sign Up" in the upper right corner.
  2. You should see a page like below, and can choose your own username, email, and password. These do not need to be affiliated with Willamette accounts if you do not want them to be, but make sure you have access to the email address.
  3. If prompted, choose the free subscription plan.
  4. If asked, feel free to give Github some info on how you plan to use the account (education probably) or just skip this step.
  5. That's it! You should be able to go ahead and sign in now.

Step 2: Accepting new assignments

Whenever there is a new assignment, lab, or project given, there will be an accompanying url provided within the instructions. Following that url will bring you to a page that describes the name of the assignment and gives you the option to accept the assignment. Accepting the assignment basically makes a copy of it in your personal Github account online.

When you accept an assignment, you will see a short loading screen as Github copies the necessary files. Once it has done so, you'll be presented with a screen which contains a link which you can click to take you directly to your newly created (and private) repository where all your work will live. Step 3: Navigating and Using your Repository

In your newly created repository you will see an exact replica of the folder structure and initial files I placed in that assignment. Generally, there will always be a few files that you can ignore (those that start with a dot). There will also always be a README file which is displayed and which describes the requirements and guidelines for that assignment. In particular, it will tell you what files you should be editing/changing, or, occasionally, adding.

Downloading assignment materials

Before you can start making edits to an assignment, you'll first need to get the assignment onto your own computer. (You CAN technically edit files online within the Github interface, but I do not recommend it. You almost assuredly will get errors on later assignments if you try to rely on the online editor.) To do so, click the green "Code" button in upper right of your repository and then select "DOWNLOAD ZIP". Once it downloads you'll need to unzip it to whatever location you like (you can't just open files from inside it! Unzip it somewhere! And make sure you know where you placed it!). Once you have things unzipped, you can fire up whatever editor you like and proceed to edit the files.

Go ahead and download the contents of today's lab to your system using the above process. Make sure you unzip the file to a location where you can find it (You can generally unzip files by either right-clicking on them in Windows or just opening the zip in MacOS).

Once you have the files on your system and unzipped, follow the instructions in the README to make the necessary changes to Info.txt. You don't need anything fancy to make the edits, as even the most simple of text editors (Notepad in Windows or TextEdit in MacOS) will work fine.

Step 4: Uploading assignment materials

Once you have made some edits to the file(s), you will need to upload them back to your online repository. The simplest way to do this is with the "Add File" button to the left of "Code" on your assignment repository page. Clicking this will then give you the option to click "Upload files". Upon clicking "Upload files" you will be brought to a new page where you can do several things:

  • Drag your desired file to upload into the space or select it through the file chooser
  • Add a short description of what you changed. A longer description can go in the space below that if necessary.
  • Commit directly to master branch

And then hit Commit changes! The folder view on Github will update immediately, so you can select the file you just uploaded and open it if you like to make sure it worked. You'll also see the number of commits increase in the upper left.

Note 1: In general you'll be editing existing files more often that uploading your own file from scratch. Uploading a file that exists already in your repository will just override the old existing file with your new version. This is what is desired! Don't worry about renaming things.
Note 2: I highly recommend to regularly upload the latest work you have back to Github, and to make meaningful commit messages when you do. The reason is two-fold:

  • Github serves as a form of version control, so if you ever really break something in your program, you can fall back to any commit you have made in the past and it will be "exactly" like you had it in that point in time. Your commit messages serves as your markers to decide what point you should roll back to. Or if your computer explodes, you'll have a perfect backup copy online.
  • Your repositories are private between just yourself and me. So at any point in time if you are having issues or struggling with something, I can easily see whatever you have uploaded. This can make it much easier to assist you remotely. I can't see things only on your local system, they need to be committed.

Upload your edited Info.txt file back to Github. Once done, click on it in the webpage to ensure that it shows your latest changes. That is it! You are done with the required portion of today! Installing Anaconda

Anaconda is a bundled version of Python that brings along not only the basic Python libraries, but also other 3rd party libraries that are frequently used and some common code editors. It's essentially a one-stop-shop for getting up and running with Python, and hence the recommended method for this semester. You can access the Anaconda webpage here, which has specific installation instructions for Windows or Mac. Following those instructions should get you what you want, but I'll make a few extra comments or explanations below:

  • When you go to download the installer (you'll probably need to scroll down the page they link you to in order to find the download links), you want the 64-Bit Graphical installer for Python 3.8.
  • You don't need to worry about the download instructions recommended action to "Verify data integrity with SHA-256". Just run the downloaded installer and install it.
  • All the other defaults or recommendations made on the webpage should be perfectly fine for our needs this semester.
  • Upon completion, launch "Anaconda Navigator" to bring up a menu from which you'll be able to launch your code editor. Spyder is installed by default, but I am really encouraging people to install VSCode this semester as it provides a much better experience in my opinion. For a guide on installing VSCode, see here.
