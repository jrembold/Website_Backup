Lab 13: List Pairs

March 10, 2021

Today is focused on building up ideas about how to define and utilize Python’s list variable type. Objectives include:

  • Defining lists
  • Performing common list operations
  • Looping over lists

Instructions

Part 1

Suppose you have a group of 10 individuals:

  • Liam, age 43
  • Emma, age 26
  • Oliver, age 16
  • Sophia, age 31
  • Elijah, age 39
  • Charlotte, age 32
  • Ben, age 19
  • Mia, age 18
  • Mason, age 24
  • Evelyn, age 34

Create two lists, one called names which is a list of all the individual’s names and one called ages which is a list of all their ages. This is just a matter of creating this lists using the above information.

Part 2

Suppose you are testing a theory that maintains that only people older than 28 have been or currently are married. Write a predicate function called predict_status, which takes the list of ages as input and loops through them, returning a list of Trues and Falses depending on if the individual is older than 28 (True here corresponding to being/having been married and False otherwise). Clearly you could create this by hand given the small group you are working with here, but frequently the lists will not be so small (or the comparisons so easy). As an example, the first few terms of your returned list should look like:

[True, False, False, True, ...]

Since Liam is over 28, Emma is younger than 28, etc.

Part 3

In truth, the marital status of your group is:

  • Liam, single
  • Emma, married
  • Oliver, single
  • Sophia, single
  • Elijah, single
  • Charlotte, married
  • Ben, married
  • Mia, single
  • Mason, single
  • Evelyn, married

Construct a new list called actual_status which contains True or False's depending on their actual marital status (True being married). Again, you are just creating a list here using the above data directly: where single corresponds to False and married to True.

Part 4

Write a function called check_predictions which takes three inputs: a list of names, a list of predicted marriage statuses, and a list of actual marriage statuses. Loop through your list of individuals and for each determine if the predicted status matches their actual married status. As you go, you want to print the results to the screen in a nicely formatted output that follows a [symbol] [name] template. We are going to take advantage of some unicode symbols here, so [symbol] in the above template should be an empty checkbox if the predicted status did not match the actual status, or a checked checkbox if the two statuses agreed. If you were to look up what the corresponding values for those two symbols were, you’d find that:

Symbol Hex ASCII Integer
☐ 0x2610 9744
☑ 0x2611 9745

So you can use chr to insert the correct symbol. When you run the check_predictions with the lists you created above as input, your screen should print something that looks like:

☐ Liam
☐ Emma
☑ Oliver
☐ Sophia
☐ Elijah
☑ Charlotte
☐ Ben
☑ Mia
☑ Mason
☑ Evelyn
