Time Dilation

Einstein’s formulas on relativity are among the great equations that changed our understanding of how the world and universe function. They are prefaced largely on the idea that the speed of light in a vaccuum must be constant everywhere. When observers start moving at large speeds, this can lead to some interesting effects, one of which is that of time dilation. The idea here is that an observer who is moving very quickly will observe an event to take longer than an observer who is not moving. Mathematically, this is described by:
tmoving=tstationary1−v2c2t_{moving} = \frac{t_{stationary}}{\sqrt{1-\frac{v^2}{c^2}}}tmoving​=1−c2v2​​tstationary​​
where tmovingt_{moving}tmoving​ is the time as measured by the moving observer, tstationaryt_{stationary}tstationary​ the time measured by the stationary observer, vvv the velocity of the moving observer and ccc the constant speed of light (which is about 3×1083\times10^83×108 meters per second).

According to the Guinness Book of World Records, the fastest time to eat a hotdog with no hands is 21.6 seconds, set by Michelle Lesco in 2018. Your goal today will be to write a function which returns the time a judge moving at some speed would have reported for the amount of time it took Michelle to eat that same hotdog.

Breaking it into steps

I’ll try to provide some structure here to help you out along the way. In the template file, I’ve already provided some small pieces of code and indicated where you should add the rest.

  • Define the known constant values in this problem. Here that would be the speed of light and the time it took Michelle to eat the hotdog from a stationary observers perspective. The convention in Python when defining constant variables to to have the variable name be entirely in uppercase.
  • Define a new function called moving_time which will take a single input: the speed of the moving judge. Remember that when defining the function, you can name this parameter whatever you like, but it helps to make it descriptive.
  • Within the function, use the above equation to compute the value of tmovingt_{moving}tmoving​, or the time the moving judge would have recorded. Frequently, when coding complex equations, it is a really good idea to break the equation up into smaller pieces that you can assign to different variables, and then you can combine them all in the end. This frequently can make the equations easier to follow, and thus easier to debug and catch mistakes. Long lines of code are frequently a bad thing (though not illegal) in Python. There is a square-root function in this equation, so you can either raise everything in the square root to the 0.5 power (which Python can natively do) or you can add import math and use its square root function (math.sqrt()).
  • As the last line of your function, ensure that you return the time that the moving judge would observe! If you don’t return anything, that information is lost!
  • Note that at the bottom of the script, in the if __name__ == '__main__': portion, I am printing off the output of the script (whatever you returned) with an arbitrary velocity initially plugged in as the input. Play around with this number a few times to see how it changes the time the moving judge would have observed. Figure out about how fast a judge would have had to be moving to measure a time twice that of the stationary judge’s recorded time. Save that time in a new constant variable called ANS, which you should include up at the top of the program near your other constants.
