Paint it Black

Open up the Lab 3 folder and then open up wall_painter.py by selecting it from the left sidebar. You'll see some scaffolding code already supplied, but allow me to explain the situation first. Your task for today is to write a program that will allow Karel to paint the interior of the walls with beepers. To do so, I have already outlined a potential decomposition that you might follow, with functions started in the wall_painter.py for each step. Essentially, the decomposition breaks down into 3 steps:

  • Move Karel initially into a corner and ready to paint a wall. Karel can start anywhere along 1st street, and can initially be facing any direction. So you'll want to rotate or move Karel however necessary to get it into a corner. This will likely require two different actions: rotating to face a desired direction and then moving into a corner.
  • Paint a single wall. Obviously, this will need to be called 4 different times to paint the 4 different walls, but writing the code to paint a single wall should easily translate to the others. Here you'll want to move along the wall, placing beepers as you go, until you reach the end of the wall. However, there is already some "paint" on some of the walls, which you do not want to double up on! So if there is already paint on a particular location on a wall, Karel should not add more.
  • Reposition between walls. When you reach the end of one wall, you'll probably need to turn and maybe reposition a bit to prepare to paint the next wall. Here is where you can do that. Remember that you don't want to double paint anything!

You will need to combine these functions together then to paint the entire room.

Some things to keep in mind:
  • Your code should work for any size of room. There are other example rooms to test against, which you can load using the left-most button on the Karel window.
  • Karel has an infinite amount of beepers initially in its bag, so you do not have to worry about running out of "paint".
  • Karel values quality work, so there should be just a single beeper along all the walls. No space should have 2 beepers (an overload of paint!).
  • Karel can start anywhere on 1st street, and can be facing any direction initially.
Some hints:
  • Tackle the functions in the order that I presented them above. First write the function to move Karel into the corner from its starting position, and then test it to make sure Karel is properly moving. Then add the function to paint a single wall and make sure that works properly. Then add the repositioning. Once that is together, you should be essentially done. But paint and reposition a few more times to get the other 3 walls.
  • If Karel runs into a wall, the program errors out. Karel doesn't appreciate being run into things! So make sure you check if there is space in front of Karel before moving.
  • Because the room can have any dimensions, you will almost assuredly need to use some while loops in both the first two functions in order to move whatever distance is necessary until you would hit a wall.

Test all the rooms!

Make sure that the code your written works, without any editing, for not only the default world but also Room2.w and Room3.w. You can test your code against those rooms by running your program to bring up the Karel window, and then immediately hitting the left-most button to load a new world and selecting either Room2.w or Room3.w. Then you can hit play to test Karel in that world.
