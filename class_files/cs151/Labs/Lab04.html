Setting the Stage

A triangle number counts the number of objects necessary to construct an equilateral triangle with a given side length. For instance, the second triangle number is 3, as that is the number of objects necessary to construct an equilateral triangle with a side length of 2.

If you extend out this logic, you get something like the image below, which graphically depicts the first 5 triangle numbers.

You may note that each new triangle number is the same as the previous triangle plus the length of the new side. As such, the n^{th} triangle number could be written as:

T_n = \sum_{k=1}^n k = 1 + 2 + 3 + \cdots + n

or, in other words, by summing up all the positive integers smaller than or equal to n. Another way of thinking about this is that your pyramid always has a base equal to the number of desired rows. Then each row above it gets 1 smaller, and you have to add everything up until you reach the top row which has only a single object.

Instructions

Now for the coding! At the top of the started function in your template for today the variable n is defined, initially being assigned the value of 2. Your task is to write code to complete the function to calculate and then print off the corresponding n^{th} triangle number. So if the value of n is 2, when you run your program it should print out the number 3. If you change the value of n at the top of your code to 3, then running your program should print out the number 6. And so on. If you want a bigger number to check, the 100th triangle number is 5050. Make sure you code can correctly compute and print that value, and then call me over to be checked off!

Some things to keep in mind:

  • This is absolutely a case where the trickiest part of the task is deciding how you use the programming tools you know of: things like loops, variables, and arithmetic, to accomplish something that can be easily described in English. Depending on how you do it, the code may be very short!
  • You will probably need a loop of some sort to gain the needed flexibility of adding up to whatever n is.
  • If using a while loop, you’ll need some sort of True/False condition, which you can get with comparison operators such as < or >.
  • Often times, instead of defining lots of variables, it can be useful to define one variable and then continually update or reassign its value each iteration of a loop.
  • Remember to print out your value at the end of the function! If you don’t print something out, you won’t see any output when you try to run your program!
