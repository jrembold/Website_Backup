Guide to Github and Github Classroom

Getting a Github account if you don’t already have one

Getting setup on Github is pretty straightforward:

  1. Navigate you way to www.github.com and click on “Sign Up” in the upper right corner.
  2. You should see a page like below, and can choose your own username, email, and password. These do not need to be affiliated with Willamette accounts if you do not want them to be, but make sure you have access to the email address.
  3. On the next page, choose the free subscription plan.
  4. On this last page, feel free to give Github some info on how you plan to use the account (or just skip this step).
  5. That’s it! You should be able to go ahead and sign in now.

Using Github Classroom

Github classroom has a central hub, located here, where you can see all the current assignments and their deadlines. In general though, this will probably not be a page you need to visit very often.

More frequently, what will happen is each week when homework or some lab is posted, there will be an accompanying url. Following that url will bring you to a page that looks something like this:

Navigating and Using your Repository

In your repository you will see an exact replica of the folder structure and initial files I placed in that assignment. There will also always be a README which describes the requirements and guidelines for that assignment.

Downloading assignment materials

To get the assignment onto your own computer, where you can edit the files. Click the green “Clone or dowload” buttom in your repository and then select “DOWNLOAD ZIP”. Once it downloads you’ll need to unzip it to whatever location you like and can then proceed to edit the files. (Of course, if you are familiar with Git and want to use clone instead, by all means use that.)

Uploading assignment materials

You are free to use whatever editor or IDE you like to write files in this class (Spyder is probably the most obvious, but if you want to use VSCode or Atom or PyCharm, you are more than welcome to). Once you have made some edits to the file though, you will need to upload it back to your repository. The simplest way to do this is with the “Upload Files” button to the left of “Clone or download”. Doing so will bring up a new page where you should do several things:

  • Drag your file or select it through the file chooser
  • Add a short description of what you changed. A longer description can go in the space below that if necessary.
  • Commit directly to master branch

And then hit commit changes! The folder view on Github will update immediately, so you can select the file if you like to make sure it worked. You’ll also see the number of commits increase in the upper left. Note 1: In general you’ll be editing existing files more often that uploading your own file from scratch. The process is the same though, and it will override the old existing file. Note 2: I highly recommend to regularly upload the latest work you have, and to make meaningful commit messages when you do. The reason is two-fold:

  • Github serves as a form of version control, so if you ever really break something in your program, you can fall back to any commit you have made in the past and it will be “exactly” like you had it in that point in time. Your commit messages serves as your markers to decide what point you should roll back to.
  • Your repositories are private between just yourself and me. So at any point in time if you are having issues or struggling with something, I can see whatever your recent commits are. This can make it much easier to assist you remotely. I can’t see things only on your local system, then need to be committed.
